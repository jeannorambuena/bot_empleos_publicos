# Modelo de servicio

## Propuesta

El radar puede ofrecerse como monitoreo personalizado de oportunidades publicas por
perfil profesional. El servicio detecta convocatorias, ordena coincidencias y ayuda
a revisar plazos; no garantiza postulaciones exitosas ni promete empleos.

## Configuracion por perfil

Cada persona requiere regiones, movilidad, palabras clave positivas, exclusiones y
umbral de alerta propios. Un radar TI puede priorizar soporte, redes, servidores,
ciberseguridad y desarrollo. Para psicologia, educacion de parvulos, contabilidad u
otras areas deben ajustarse vocabulario, exclusiones y fuentes pertinentes antes de
publicar recomendaciones.

## Despliegue inicial

Para una operacion personal o comunitaria acotada:

- GitHub Pages publica un dashboard estatico.
- GitHub Actions ejecuta capturas revisadas.
- Telegram entrega previews y, solo si se habilita deliberadamente, alertas.
- GitHub conserva versionado, historial de cambios y rollback.

La version gratuita sirve para pocos perfiles, volumen bajo y revision humana
frecuente. No reemplaza soporte, monitoreo de disponibilidad ni una base privada.

## Cuando migrar

Conviene pasar a hosting propio cuando existan varios usuarios, perfiles privados,
mayor frecuencia de captura, necesidad de autenticacion, acuerdos de servicio,
metricas, colas de trabajo o integraciones con datos personales.

## Privacidad

No publiques RUN/RUT, nominas, nombres de postulantes, contactos innecesarios ni
resultados historicos sensibles. Mantener sanitizacion, revision humana y separacion
entre artefactos locales y `public/` es obligatorio. Cada nueva fuente debe entrar
como candidata o dry-run antes de cualquier promocion.

## Limites comerciales

El servicio ofrece monitoreo, trazabilidad y priorizacion asistida. La persona
usuaria sigue siendo responsable de revisar bases, requisitos, fechas y canal
oficial de postulacion.
